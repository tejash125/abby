# Building Blocks — rough working notes

Status: **rough notes, thinking in progress.** This is our own assumed thinking
(not summarized from outside), building on `README.md`. Companion research notes
live in `research/representation-and-conventions.md` and
`research/foundational-blocks-universe-and-abby.md`.

Goal of this file: pin down *exactly* what Abby needs built in, what it must grow
on its own, and how the very first non-conventional representations could look —
especially time and decay, without using any language or number system to say it.

---

## The core distinction: lived facts vs told facts

- **Told fact** — arrives through a convention (language, symbols, numbers).
  Needs a shared code and trust *first*. Cannot be foundational, because a fresh
  learner has no code to receive it through.
- **Lived fact** — a condition Abby is *subject to*, built into how it works,
  felt directly, never encoded. Abby feels it because it changes what Abby can
  do.

**Consequence:** we never *tell* Abby a foundation. We **instantiate** conditions;
Abby **represents** them. We install decay; Abby grows the idea of decay. We open
a channel; Abby grows the code. We (researchers/world) can only ever act on what
Abby senses — we are a presence in its world, not a narrator to it.

**Hard constraint that follows:** every built-in condition must be *feelable from
the inside*. A decay that drains in hidden bookkeeping is invisible and
unlearnable. It has to couple into Abby's own signals (predictions get noisier,
actions weaken, a background internal signal shifts) so the predict-correct loop
can catch it. If Abby can't feel it, it isn't there.

---

## The list: built-in seeds vs emergent (grown by Abby)

Design discipline: **the built-in list must be as short as we can make it.**
Everything on the emergent list must NOT be installed. Every item we can move
from built-in to emergent is a win. If we catch ourselves installing something
from the emergent list, stop.

### Built-in seeds (instantiated, never "represented" by us)

1. **A patterned world** — an environment with regularities, or there is nothing
   to learn.
2. **A boundary** — something that separates Abby from not-Abby.
3. **Sensation** — continuous channels where the world pushes on Abby.
4. **A lasting trace (memory)** — so the past can shape the next guess.
5. **The predict-correct loop** — expect next, feel mismatch, adjust. Self-graded.
   (Engine 1.)
6. **A continuous representational currency** — analog values and relations, NOT
   discrete tokens/symbols/integers. (See representation section; this is a
   substrate choice, flagged as possibly-too-much below.)
7. **The decay seed** — an internal quantity that constantly drains; if it hits
   zero, Abby stops; the drain can be slowed by predicting/acting well; and the
   drain is *feelable from the inside*.
8. **Action** — a way to affect the world and its own state (needed to slow decay,
   and later to signal).
9. **A channel to another agent** — a *medium* (not a code) over which signals can
   pass both ways, so conventions can emerge. (Possibly just more of #3 + #8.)

### Emergent (Abby builds these; we must not install them)

- A representation of **time / duration** — from felt one-way decay + predicting
  the next moment.
- The concept of **decay / aging / death** — from felt decline + observing other
  things stop.
- **Relevance** — what moves the budget.
- **Goals / drives** — surviving Abbys behave *as if* "don't hit zero"; selection
  installs the drive, not us.
- **Trust** — a source Engine 1 finds consistent and need-meeting.
- **Conventions / shared codes** — from coordination games over the channel.
- **Concepts / categories / "things"** — stable clusters of predictors that move
  together.
- **Understanding** — a convention anchored to a self-caught regularity.
- **Calendars / clock-reading** — external cycles mapped onto internal felt
  duration. (The P0 target.)

Note how much moved to the right. Time, goals, relevance, and clock-reading are
all *grown from the single decay seed* (#7) plus the predict-correct loop (#5).

## The original sensory representation, and how it evolves

**What a representation is, before any convention:** an internal state that
(a) moves along with something Abby is subject to (a sensation or an internal
condition), and (b) feeds the next-moment prediction. That's it. Not a symbol,
not a label, not a count.

**Why it must be continuous / analog / relational, not discrete:** counting,
integers, and symbols are *conventions humans invented* — we must not smuggle
them in as Abby's native currency (this is the "avoid discrete math" pitfall the
project flagged). A baby doesn't hold "3 apples"; it has continuous more/less,
near/far, brighter/dimmer. So Abby's native currency is **continuous values and
the relationships between them**, with the next-moment built in (every value is
partly a prediction of what comes next).

**First form:** a field of continuous internal quantities, each a small
predictor's current estimate, each related to others, all aimed at the next
moment. Time is not a stored axis — it *is* the fact that predictions are about
what comes next. Decay is not a stored number — it *is* a continuous internal
quantity that drains and is felt.

**How it evolves:**
1. Prediction error carves the relationships. Predictors that reduce surprise get
   reinforced/stabilized; ones that don't fade.
2. Clusters of predictors that reliably move together harden into stable units —
   proto-"things," proto-categories. (This is where "objects" first appear, as
   knots of co-moving values — the relational/structural view from the physics
   research, one level up.)
3. Only much later, when a second agent is present, do coordination games layer
   *conventions* on top of this grounded base.

**How time and decay specifically get represented (the answer to the original
question):** we never store "there is time" or "there is decay." We build the
decay in as a felt, draining, one-way internal quantity. Because it only goes one
way on its own, it gives Abby a *private arrow* — an internal signal where "later"
reliably means "less left." Abby's predict-correct loop catches this regularity in
itself and *builds* a representation of duration from it. The felt drain is the
raw material; the concept of time is the learned structure over it. Aging is the
long integral of the fast drain; the concept of death is a later generalization
from felt decline plus watching other things stop.

## Minimal conventions — build the story with as few pitfalls as possible

**Why conventions exist:** to coordinate between agents. Before imposing any of
ours, ask: what is the *most minimal* shared code two agents can form?

From the signaling-game / naming-game research: it's *any* signal both come to
associate with the same state through repeated successful coordination. It need
NOT be discrete words. It can be a continuous gesture, a matched internal state, a
shared rhythm. Discreteness (separate words, integers) is itself a later
convention, not a starting requirement.

**The rule that keeps us minimal:** we provide the **medium**, never the **code.**
- We open a channel (a way for Abby to emit something another agent can sense, and
  vice versa).
- We do NOT pre-load English, or integers, or any symbol set.
- A shared code *emerges* from coordination games, grounded in what both agents
  actually sense and in the decay-stakes that make coordinating worthwhile.

**Pitfalls to avoid (the "don't smuggle conventions in" list):**
- Don't hand Abby a language (English, or any designed token language). That skips
  grounding — Abby would recite, not understand.
- Don't hand Abby discrete math / integers as its native representation. Its native
  currency is continuous and relational; number is a convention to be *earned*
  later, not assumed.
- Don't build a reward number and call it a goal. The goal must emerge from decay +
  selection.
- Don't build a clock. Time must emerge from felt decay + prediction.
- Don't narrate to Abby. Even *we* must interact only by shaping its sensory world;
  our channel to Abby is convention-free at the start, same as another agent's.

**What "a better way to talk with Abby" looks like:** not a language we design, but
a grounded, continuous, two-way sensory channel over which a shared code
self-organizes — the same predict-correct loop, now running between agents. The
first "words" will be whatever continuous signals reliably help two Abbys (or Abby
and us) coordinate to slow their decay.

## Open tensions to keep pulling on

- **Is #6 (continuous currency) a convention we're imposing, or a fair substrate
  choice?** It might be the one representational assumption we can't avoid — but we
  should be honest it's an assumption, and check whether even continuity should be
  something Abby's medium *exhibits* rather than something we declare.
- **Is #9 (channel) a separate primitive, or just sensation + action pointed at
  another agent?** Leaning toward the latter — fewer seeds is better.
- **One decay or two?** A fast drain (powers felt now→next and urgency) vs a slow
  lifetime drain (powers aging, lifespan, calendars). Hunch: build the fast one,
  let aging emerge as its long sum. Not resolved.
- **Does Abby need peers to get the concept of death?** You learn "everything that
  runs down eventually stops" by watching others stop while you continue, plus
  feeling your own decline. That argues for a multi-agent world from early on —
  which is also where conventions can form at all.
- **Memory (#4) is doing a lot of quiet work.** "A lasting trace" is vague. What is
  the minimal form of memory that lets prediction work, without smuggling in
  storage-as-symbols? Needs its own pass.
